Process Scheduling

Process scheduling is an important task of the OS that ensures the best utilization of the CPU. Some of the process scheduling algorithms are - First-Come First Served, Shortest Job First, Shortest Remaining Time First, Priority, Round Robin and Multilevel Queue scheduling.

FCFS

First Come First Served

First Come First Serve (FCFS) is an operating system scheduling algorithm that automatically executes queued requests and processes in order of their arrival.. It is rarely used in modern operating systems.

Go
SJF

Shortest Job First

SJF is a scheduling policy that selects the waiting process with the smallest execution time to execute next(non-preemptively). It is optimal but practically infeasible scheduling algorithm.

Go
SRTF

Shortest Remaining Time First

SRTF is a scheduling method that is a preemptive version of SJF scheduling. In this scheduling algorithm, the process with the smallest amount of time remaining until completion is selected to execute.

Go
Priority

Priority Scheduling

Priority scheduling is a method of scheduling processes based on priority. Same priority processes are executed on FCFS basis. Commonly used in batch systems.

Go
RR

Round Robin Scheduling

Round Robin Scheduling is a CPU scheduling algorithm that assigns CPU on basis of FCFS for fixed time called as time quantum.Once a process is executed for a given time period, it is preempted and other process executes for a given time period. .

Go
MLQS

Multilevel Queue Scheduling

A multi-level queue scheduling algorithm partitions the ready queue into several separate queues. The processes are permanently assigned to one queue, each with an assigned priority and its own scheduling algorithm. For the process in a queue to execute, all the queues of higher priority should be empty.
