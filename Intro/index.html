Deadlock

Banker's Algorithm for Deadlock Avoidance - Each time a process enters the system, as part of the Resource-Request Algorithm, we pretend to allocate requested resources to the process by modifying the state and then run the Safety Algorithm. If resulting state is safe, the resources are allocated to the process. Otherwise, the process must wait and the old resource-allocation state is restored, thereby avoiding deadlock.
